## Git worktree

Все фичи разрабатываются через отдельный git worktree.

## Правила
- Всегда использовать `git worktree` (запрещено `git checkout` для создания фич)
- Базовая ветка: `main`
- Рабочая директория основного проекта не должна использоваться для разработки фич (все изменения только в worktree)

## Нейминг
- Ветка: `features/<id>-<n>`
- Директория: `../features-<id>-<n>`
- `<id>-<n>` — единый идентификатор фичи

## Процесс
Создание worktree обязательно перед стартом SDD workflow.

Идентификатор `<id>-<n>` является единым для:
- ветки
- worktree директории
- всей SDD документации

## Workflow (SDD)

Один цикл = одна фича. Запрещено переходить к следующему этапу без явного подтверждения готовности текущего.

Полные детали, уровни важности и критерии завершения — в docs/workflow.md.

1. **Бриф** — из GitHub issue, копируется в `brief.md`
2. **Ревью брифа**
3. **Спецификация**
4. **Ревью спецификации** — `/spec-reviewer:spec-review`
5. **План реализации**
6. **Ревью плана**
7. **Реализация**
8. **Приёмка** — тесты, PR, завершение цикла

## Документация

SDD артефакты создаются по мере выполнения workflow.

## Путь
docs/features/<id>-<n>/

## Правила
- структура соответствует `<id>-<n>` из ветки `features/<id>-<n>`
- файлы создаются строго по этапам SDD workflow (запрещено создавать артефакты заранее)
- каждый этап создаёт только необходимые для него документы
- запрещено пропускать этапы workflow или объединять их
- после завершения приёмки допускается удаление worktree (если не требуется для отладки)

## Стек
Ruby on Rails 8.1.3, PostgreSQL, RSpec  
Phlex через phlex-rails для вью и лейаутов  
Superform для форм  

## Соглашения
- Стандартный Rails MVC, пока без service objects
- RSpec для тестов, FactoryBot для фикстур
- Новые гемы предлагаются на этапе плана и требуют явного одобрения до реализации
- Никаких `.erb` — только Phlex-компоненты
- Лейауты: `app/views/layouts/` как классы Phlex
- Компоненты: `app/views/components/`
- Именование: `Users::IndexView`, `Users::ShowView` и т.д.
- Формы: только Superform (без form_with / form_for)

## Ключевые команды

- `bin/setup` — bootstrap
- `bin/rails s` — запуск сервера
- `bundle exec rspec` — запуск тестов
- `bin/rails db:migrate` — миграции

## Ограничения

- Не трогать существующие миграции
