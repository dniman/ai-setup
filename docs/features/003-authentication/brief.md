# Бриф: Аутентификация пользователей

**Issue:** #3

## Что делаем

Реализуем аутентификацию на основе `bin/rails generate authentication`: форму входа по email и паролю, поддержание сессии (хранится в БД), выход и восстановление пароля по email. После внедрения все существующие страницы требуют входа — неаутентифицированные пользователи перенаправляются на форму логина. После успешного входа — редирект на root_path.

Для тестирования создаётся seed-пользователь (без роли) через `db/seeds.rb`.

Публичная регистрация отсутствует — это сознательное ограничение. Создание аккаунтов администратором, разграничение ролей и блокировка при неудачных попытках входа — отдельные задачи, не входят в scope.

## Зачем

Сейчас система полностью открыта — любой может получить доступ к любому разделу. Аутентификация нужна как фундамент для дальнейшего разграничения доступа и защиты данных. Без неё невозможно отличить одного пользователя от другого.
